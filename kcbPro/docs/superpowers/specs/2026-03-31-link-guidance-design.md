# Link Guidance Design

**Date:** 2026-03-31

## Goal

在 token 课表页中加入“首次展示完整访问链接一次”的引导，并统一电脑端与手机端的链接找回入口：

- 默认显示本周课程
- 同一个 token 首次访问时自动展示完整链接提示条
- 用户复制成功后，提示条延迟几秒自动消失
- 后续再访问时不再自动展示
- 桌面端通过右上角弱化的“设置”按钮进入复制入口
- 手机端通过底部导航“我的 / 设置”进入复制入口

## Scope

本次设计只覆盖以下能力：

- token 页首次访问状态记录
- 默认周次调整为当前周
- 首次链接提示条显示/隐藏逻辑
- 设置入口在桌面端与手机端的布局差异
- 设置层中的“复制当前链接”动作

本次不包含以下能力：

- 多个设置项的复杂设置页
- token 过期/回收
- 链接分享统计
- 设备级首访记录

## Product Decision

“只显示一次”按 **每个 token 的首次访问** 计算，而不是按设备计算。

这意味着：

- 某个 token 第一次被打开时显示完整链接提示条
- 一旦用户在这次展示中完成复制，服务端将该 token 标记为“已展示”
- 之后同一个 token 再访问时，不再自动展示
- 如果用户仍需要复制链接，则通过“设置”入口完成

## User Flow

### First Visit

1. 用户访问 `/t/<token>`
2. 页面默认进入当前周课程
3. 若该 token 尚未展示过首次提示，则页面顶部显示完整链接提示条
4. 用户点击“复制当前链接”
5. 浏览器复制成功
6. 前端调用后端接口，将该 token 标记为“首次提示已展示”
7. 提示条在复制成功后延迟几秒自动消失

### Later Visits

1. 用户再次访问 `/t/<token>`
2. 页面直接显示本周课程，不再自动展开完整链接提示条
3. 若用户需要复制链接：
   - 电脑端点右上角“设置”
   - 手机端点底部导航“我的 / 设置”
4. 在设置层中点击“复制当前链接”

## Interaction Design

### Desktop

- 顶部右侧显示弱化样式的“设置”按钮，不使用高强调 CTA
- 首次提示条位于页面头部下方，优先级高于课表但不遮挡整个界面
- 设置按钮点击后，打开轻量菜单或抽屉
- 设置层中包含：
  - `复制当前链接`
  - `重新显示保存提示`（可选，便于用户再次查看）

### Mobile

- 课表页首次进入时，顶部同样展示完整链接提示条
- 后续复制入口放在底部导航“我的 / 设置”
- 点击后打开底部抽屉或设置面板
- 设置面板中包含：
  - `复制当前链接`
  - `重新显示保存提示`（可选）

## Data Model Changes

现有 `saved_logins` 表需要增加一个字段：

- `link_hint_seen INTEGER NOT NULL DEFAULT 0`

语义：

- `0` 表示该 token 还未完成首次链接提示流程
- `1` 表示该 token 已完成首次复制提示，不再自动展示

## Route/API Changes

### Existing Page Route

`GET /t/<token>`

新增模板变量：

- `show_link_hint`
- `saved_link`
- `settings_entry_mode`

其中：

- `show_link_hint` 由 `link_hint_seen` 决定
- `saved_link` 应使用 canonical token 路径，而不是直接复用 `request.url`

### New API

新增一个专门的状态写回接口，例如：

- `POST /api/tokens/<token>/link-hint-seen`

行为：

- token 不存在：返回 404
- 成功：将 `link_hint_seen` 更新为 `1`
- 返回 `{ "ok": true }`

## Frontend Behavior

### Default Week

- token 页面默认应显示当前周
- 不再让“全部课程”作为默认已选状态
- 如果仍保留“全部课程”选项，则该选项必须真实可用

### First-Time Banner

提示条应包含：

- 说明文案：
  - “登录成功。请保存此链接，下次可直接访问。”
- 完整链接展示
- `复制当前链接` 按钮

复制成功后：

- 显示轻量成功反馈
- 调用“标记已展示”接口
- 延迟几秒后自动收起

复制失败后：

- 提示“复制失败，请手动复制”
- 不自动收起

### Settings Entry

- 电脑端：右上角弱化“设置”按钮
- 手机端：底部导航“我的 / 设置”
- 设置入口中的视觉权重应低于首次提示条

## Error Handling

- token 无效：沿用现有“链接无效，请重新登录”
- 复制失败：提示手动复制，不隐藏提示条
- 标记已展示接口失败：
  - 当前复制仍然算成功
  - 但由于服务端状态未写入，下次可能还会再次展示
- 设置层打开时若 token 丢失：显示不可复制状态

## Testing Strategy

至少覆盖以下测试点：

- 首次访问 token 页面时，模板渲染 `show_link_hint = true`
- 已标记的 token 再访问时，不自动展示提示条
- 标记接口能正确把 `link_hint_seen` 更新为 `1`
- 页面默认周次为当前周
- 设置入口在模板中存在
- 复制当前链接使用 canonical token 路径

## Risks And Tradeoffs

- 按 token 维度记录首次展示，意味着只要有人先复制过，这个 token 以后对所有访问者都不再自动展示
- 若用户复制成功但写回状态失败，下次会再次看到提示条
- 手机端“我的 / 设置”入口更符合 App 习惯，但实现比单按钮更复杂

## Final Design Summary

最终交互方案如下：

- 默认进入本周课程
- token 首次访问自动显示完整链接提示条
- 用户复制成功后，几秒后自动收起
- 服务端将该 token 标记为已展示
- 之后不再自动展示
- 桌面端通过右上角弱化“设置”按钮复制当前链接
- 手机端通过底部导航“我的 / 设置”复制当前链接
